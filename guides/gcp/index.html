Deployment Models

The guide covers two GCP deployment models. Both use VPC Peering for hub-spoke connectivity and the official Palo Alto Networks Terraform modules for infrastructure provisioning.

ModelFirewallsLoad BalancersBest For
Common-Firewall 2 (single set handles all traffic) Internal NLB + External Regional NLB Most deployments, POC, moderate traffic
Dedicated-Inbound 4 (separate inbound and outbound/east-west sets) Internal NLB + Global HTTP LB High inbound traffic, production scale
GCP Architecture

Unlike AWS (GWLB) and Azure (Gateway LB), GCP uses an ILB sandwich architecture — External NLB for inbound traffic, Internal NLB for outbound and east-west traffic, with VM-Series firewalls as backends. This requires management interface swap because GCP's Internal Load Balancer can only forward traffic to nic0.
